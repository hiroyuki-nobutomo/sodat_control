firstrun スニペット ダウンロード

サービスアカウントキーは Vercel 側で保管済みです (環境変数 SODAT_SERVICE_ACCOUNT_JSON_B64)。研究者は鍵ファイルを持ち歩く必要がなく、ユーザ名を入力してダウンロードするだけです。

1. Pi Imager で設定するユーザー名

2. インストールするセンサーを選択

この Pi に物理的に接続するセンサーだけを残してください。未選択のセンサーは config.yaml から除外され、ログにエラーが出ません。デフォルトは全選択。

snippet をダウンロード

3. SD カードに貼り付ける

  1. microSD を PC に挿し、bootfs パーティションを開く
  2. firstrun.sh をテキストエディタで開く (Win: メモ帳、Mac: テキストエディット の標準テキストモード)
  3. ファイル末尾の rm -f /boot/firstrun.sh (もしくは exit 0) の直前に、ダウンロードした sodat-firstrun-snippet.sh の本体行 (# >>> BEGIN ... から # <<< END までの行) を貼り付ける
  4. 保存 (改行コードは LF のまま、CRLF に変えない)
  5. SD を取り出して Raspberry Pi に挿入 → 電源 ON
ダウンロードした .sh ファイルにはプロジェクトの service_account.json (base64) が含まれます。チャットやメール本文に貼らないでください。SD へ書き終わったら PC 上のダウンロードファイルも削除推奨。
仕組み: 鍵がどこにあるか

Vercel 上の Serverless Function (/api/firstrun) が、環境変数の SA キーとリクエストの user パラメータを firstrun_snippet.sh.template に差し込み、完成済みの .sh を返します。このページはアクセス制限下にあるため (Vercel Deployment Protection)、URL を知っているだけでは鍵を取り出せません。

firstrun snippet download

The service-account key is stored on Vercel as an environment variable (SODAT_SERVICE_ACCOUNT_JSON_B64). You don't need to carry the key file around — just enter the Pi Imager username and download.

1. The Pi Imager username

2. Pick sensors to install

Check only the sensors physically connected to this Pi. Unchecked sensors are removed from config.yaml so they don't generate runtime errors. Defaults to all sensors selected.

Download snippet

3. Place on the SD card

  1. Insert the microSD into your PC; open the bootfs partition.
  2. Open firstrun.sh in a text editor (Notepad on Windows, TextEdit in plain-text mode on macOS).
  3. Near the end you'll find rm -f /boot/firstrun.sh (or exit 0). Immediately above that, paste the body of the downloaded sodat-firstrun-snippet.sh (everything between the # >>> BEGIN ... and # <<< END markers).
  4. Save the file. Keep line endings as LF (don't let your editor switch to CRLF).
  5. Safely eject the SD card, insert it into the Pi, power on.
The downloaded .sh file contains the project's service_account.json (base64). Don't paste it into chat or email. Delete the downloaded copy from your PC after the SD card is written.
How it works: where the key lives

A Vercel Serverless Function at /api/firstrun injects the environment-variable SA key and the request's user parameter into firstrun_snippet.sh.template and returns a finished .sh. This page is behind Vercel Deployment Protection, so simply knowing the URL doesn't expose the key.
