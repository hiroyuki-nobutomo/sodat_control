📂 ボタンは、SD カードの bootfs フォルダをブラウザが直接読み書きします。ブラウザの権限ダイアログで「許可」を選んでください (このタブを閉じれば失効)。 📂 lets the browser read and write the SD card's bootfs folder directly. Grant permission when the browser asks (revoked when this tab closes).

方法 A (推奨): 📂 SD カードに直接書き込む

  1. microSD を PC に挿す (Mac なら自動で bootfs ボリュームがマウント)
  2. 上の 📂 ボタンをクリック → フォルダ選択ダイアログで bootfs を選ぶ
  3. ブラウザの「許可」を押す
  4. 「✓ SD に書き込み完了」を確認したら SD を取り出して Raspberry Pi に挿入 → 電源 ON

※ Chrome / Edge / Safari 17.4 以降で対応。古い Safari / Firefox では 📂 ボタンが表示されないので、下の方法 B を使ってください。

方法 B (手動): 📋 Copy + テキストエディタでペースト

  1. microSD を PC に挿し、bootfs パーティションを開く
  2. firstrun.sh をテキストエディタで開く (Win: メモ帳、Mac: テキストエディット の標準テキストモード)
  3. ファイル末尾の rm -f /boot/firstrun.sh (もしくは exit 0) の直前に、📋 Copy で取得した snippet を貼り付ける
  4. 保存 (改行コードは LF のまま、CRLF に変えない)
  5. SD を取り出して Raspberry Pi に挿入 → 電源 ON

Method A (recommended): 📂 Write directly to the SD card

  1. Insert the microSD into your PC (on macOS the bootfs volume mounts automatically).
  2. Click the 📂 button above → in the folder picker, select bootfs.
  3. Click Allow on the browser permission prompt.
  4. When you see "✓ Wrote firstrun.sh", eject the SD card, insert it into the Pi, and power on.

Supported in Chrome / Edge / Safari 17.4+. On older Safari / Firefox the 📂 button is hidden; use Method B below.

Method B (manual): 📋 Copy + paste in a text editor

  1. Insert the microSD into your PC; open the bootfs partition.
  2. Open firstrun.sh in a text editor (Notepad on Windows, TextEdit in plain-text mode on macOS).
  3. Near the end you'll find rm -f /boot/firstrun.sh (or exit 0). Immediately above that, paste the snippet you grabbed with 📋 Copy to clipboard.
  4. Save the file. Keep line endings as LF (don't let your editor switch to CRLF).
  5. Safely eject the SD card, insert it into the Pi, power on.
ダウンロード / コピーした snippet にはプロジェクトの service_account.json (base64) が含まれます。チャットやメール本文に貼らないでください。書き終わったら PC 上のダウンロードファイルも削除推奨。 The snippet (downloaded or copied) embeds the project's service_account.json (base64). Don't paste it into chat or email. Delete the downloaded copy from your PC once the SD card is written.

仕組み: 鍵がどこにあるか How it works: where the key lives

Vercel 上の Serverless Function (/api/firstrun) が、環境変数の SA キーと URL の user / sensors / token を firstrun_snippet.sh.template に差し込み、完成済みの .sh を返します。アクセストークン検証は node:crypto.timingSafeEqual による定数時間比較。 A Vercel Serverless Function (/api/firstrun) injects the env-var SA key and the URL's user / sensors / token into firstrun_snippet.sh.template and returns a finished .sh. Token check is constant-time via node:crypto.timingSafeEqual.
