農業タスク動的マッチング — 3つの画面

標準作業辞書（task_types 65タスク）を中核に、タスクを軸として実績・スキル・求人・マッチングが連携します。データはダミー（fixtures.json）。TPOCast連携は後日。

🚜

農園管理者用

栽培状況からの次タスク提案、求人の作成、候補者マッチング結果の確認。
🧑‍🌾

短期バイト応募者用

自分のスキル・資格・対応可能時間、届いたオファーの承諾／辞退。
📋

作業者用

実施した作業を記録（実績蓄積→スキル自動育成）。TPOCastの役割を代替。

データフロー

作業者が実績を記録 → スキルが育つ ／ 栽培状況から次タスクを提案 → 農園が求人 → エンジンがタスク軸でマッチ → 応募者にオファー。
