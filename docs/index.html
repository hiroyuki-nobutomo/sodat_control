🌱Sodat Sensor SFC — SD カードセットアップ

農業・園芸現場のセンサーデータを Raspberry Pi で集めて Google Drive へ

このページの手順通りに 1 枚の microSD を作れば、Raspberry Pi に挿して電源を入れるだけでセンサ収集が始まり、Google Drive / Sheets へのアップロードまで動きます。SSH や scp は不要です。

用意するもの

  • Raspberry Pi (Zero 2 W / 3 / 4 / 5 のいずれか)
  • microSD カード (8 GB 以上推奨)
  • SD カードリーダ付きの PC (Windows / macOS / Linux いずれも可)
  • プロジェクト管理者から受け取った service_account.json (Google サービスアカウント鍵)

手順 1. Raspberry Pi Imager をインストール (1 回だけ)

Raspberry Pi Imager をダウンロード

各 OS のインストーラに従ってインストールしてください。

手順 2. Pi Imager で SD カードを書き込み

  1. Pi Imager を起動し、左の 「デバイスを選ぶ」 で使う Raspberry Pi の機種を選ぶ。
  2. 真ん中の 「OS を選ぶ」 → Raspberry Pi OS (other) → Raspberry Pi OS (64-bit) を選ぶ (フル版推奨。より軽量にしたい場合は Raspberry Pi OS Lite (64-bit) でも動作します)。
  3. 右の 「ストレージを選ぶ」 で microSD を選ぶ。
  4. 「次へ」を押すと「OS をカスタマイズしますか?」と聞かれるので 設定を編集する を選ぶ。
  5. 「一般」タブで次を設定:
    • ホスト名 = 機器名: s01, s02 ... (SFC に設置する場合) / a01, a02 ... (API 機構に設置する場合)。機器ごとに必ず変える。Pi Imager 側は小文字で入力 (DNS 規約)、初回起動で大文字化されて device_id (S01 / A01) に自動反映される
    • ユーザー名とパスワード: 任意 (例: ユーザー sodat)。後で SSH する可能性があるなら覚えておく
    • 無線 LAN: SSID / パスワード / 国 (JP) を設定
    • ロケール: タイムゾーン Asia/Tokyo
  6. 「サービス」タブで SSH を有効化 (パスワード認証で可)。
  7. 保存 → はい で書き込み開始。書き込み + 検証で 5〜10 分。

手順 3. firstrun snippet を SD カードに乗せる

書き込みが終わったら microSD を一旦取り出し、もう一度 PC に挿し直してください。bootfs という名前のパーティションが見えるはずです。

下のページに進み、Pi Imager で設定したユーザー名を入力 → 「snippet をダウンロード」 をクリック → ダウンロードされた sodat-firstrun-snippet.sh の本体行を bootfs/firstrun.sh に貼り付けるだけです。service_account.json は Vercel 側で管理されているため、研究者の PC では一切扱いません。

firstrun snippet ダウンロードページを開く →

取り扱い注意: ダウンロードされた .sh ファイルにはプロジェクトの service_account.json が base64 で含まれます。チャットや公開リポジトリに貼らないでください。SD への書き込みが終わったら PC 上のダウンロードファイルも削除推奨。

手順 4. Pi に挿して電源 ON

bootfs をアンマウント / 取り出して、microSD を Raspberry Pi に挿してください。電源を入れると初回起動が始まり、自動で:

  1. Wi-Fi に接続 (Pi Imager の設定)
  2. service_account.json を所定位置に展開
  3. GitHub からリポジトリを取得しインストール
  4. ホスト名から device_id を導出 (s05 → S05、a01 → A01)
  5. センサ収集サービスを起動

所要時間は 8〜15 分程度。インターネット回線次第です。

動作確認

同じネットワーク上の PC から:

ssh <ユーザー名>@sNN.local      # 例: ssh sodat@s05.local
sudo systemctl status sensor_sfc

active (running) になっていれば成功です。ログは journalctl -u sensor_sfc -f で追えます。

トラブルシュート

  • 起動しても Google Drive にデータが来ない: /var/log/sodat-firstrun.log を確認。journalctl -u sensor_sfc -n 200 も参照。
  • sNN.local / aNN.local で繋がらない: ルータの管理画面で IP を確認、または Pi にディスプレイを繋いで ip a で確認。
  • もう一度やり直したい: SD を PC で再フォーマットして手順 2 から。

🌱Sodat Sensor SFC — SD Card Setup

Field-sensor data collection for agriculture & horticulture, on a Raspberry Pi → Google Drive

Follow the steps below to prepare one microSD card. Insert it into a Raspberry Pi, power on, and sensor collection + Google Drive / Sheets upload starts automatically — no SSH or scp required.

What you need

  • A Raspberry Pi (Zero 2 W / 3 / 4 / 5)
  • A microSD card (8 GB+ recommended)
  • A PC with an SD-card reader (Windows / macOS / Linux)
  • The project's service_account.json (Google service-account key — obtain from your project lead)

Step 1. Install Raspberry Pi Imager (one-time)

Download Raspberry Pi Imager

Step 2. Write the SD card with Pi Imager

  1. Open Pi Imager. Under Choose device, pick your Pi model.
  2. Under Choose OS → Raspberry Pi OS (other) → Raspberry Pi OS (64-bit) (full version recommended; Raspberry Pi OS Lite (64-bit) also works for a lighter footprint).
  3. Under Choose storage, pick the microSD card.
  4. Click Next. When asked "Would you like to apply OS customisation settings?", choose Edit Settings.
  5. On the General tab:
    • Hostname = device name: s01, s02, ... (devices at SFC) or a01, a02, ... (devices in the API building). Must be unique per device. Enter in lowercase (DNS rules); first boot uppercases it into the device_id (S01 / A01).
    • Username and password: e.g. sodat. Remember these — you may need them for SSH later.
    • Wireless LAN: SSID / password / country.
    • Locale: set the timezone (e.g. Asia/Tokyo).
  6. On the Services tab: enable SSH (password auth is fine).
  7. Click Save → Yes to start writing. Write + verify takes 5–10 minutes.

Step 3. Place the firstrun snippet on the SD card

When writing finishes, eject and re-insert the microSD on your PC. You should see a partition called bootfs.

On the next page, enter the Pi Imager username and click Download snippet. The downloaded sodat-firstrun-snippet.sh body is pasted into bootfs/firstrun.sh. The service_account.json is managed centrally on Vercel — researchers don't handle the key file themselves.

Open the firstrun snippet download page →

Handle with care: the downloaded .sh file embeds the project's service_account.json (base64). Don't paste it into chat or a public repo. Delete the downloaded copy from your PC after the SD card is written.

Step 4. Boot the Pi

Unmount/eject bootfs, insert the microSD into the Raspberry Pi, and power on. The first boot will automatically:

  1. Connect to Wi-Fi (from Pi Imager settings).
  2. Place service_account.json at the right location.
  3. Clone the repo from GitHub and run the installer.
  4. Derive device_id from the hostname (s05 → S05, a01 → A01).
  5. Start the sensor-collection service.

This takes ~8–15 minutes depending on your internet speed.

Verify

From a PC on the same network:

ssh <username>@sNN.local      # e.g. ssh sodat@s05.local
sudo systemctl status sensor_sfc

Look for active (running). Logs: journalctl -u sensor_sfc -f.

Troubleshooting

  • Service running but no data in Drive: check /var/log/sodat-firstrun.log and journalctl -u sensor_sfc -n 200.
  • Can't reach sNN.local / aNN.local: find the Pi's IP via your router, or connect a display and run ip a.
  • Want to start over: reformat the SD card on your PC and repeat from Step 2.
