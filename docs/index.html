Pi Imager で SD カードを書き込みWrite the SD card with Pi Imager

Pi Imager v2.0 以降が必要です (v1.x は firstrun.sh 方式、v2.0+ は cloud-init user-data 方式)。このウィザードは v2.0+ 専用です。古い場合は 公式サイト から最新版を再インストールしてください。

Pi Imager v2.0 or later is required. (v1.x writes firstrun.sh; v2.0+ writes cloud-init user-data.) This wizard targets v2.0+. If you have an older version, reinstall from the official site.

  1. Pi Imager を起動し、左の 「デバイスを選ぶ」 で使う Raspberry Pi の機種を選ぶ。
  2. 真ん中の 「OS を選ぶ」 → Raspberry Pi OS (other) → Raspberry Pi OS (64-bit) を選ぶ (フル版推奨。より軽量にしたい場合は Raspberry Pi OS Lite (64-bit) でも動作します)。
  3. 右の 「ストレージを選ぶ」 で microSD を選ぶ。
  4. 「次へ」を押すと OS カスタマイズの確認画面が出るので 設定を編集する を選ぶ。
  5. 「Hostname」を設定:
    • ホスト名 = 機器名: s01, s02 ... (SFC に設置する場合) / a01, a02 ... (API 機構に設置する場合)。機器ごとに必ず変える。Pi Imager 側は小文字で入力 (DNS 規約)、初回起動で大文字化されて device_id (S01 / A01) に自動反映される
  6. 「User」を設定:
    • ユーザー名とパスワード: 任意 (例: ユーザー sodat)。次のステップで同じユーザー名を入力するので覚えておく
  7. 「Wi-Fi」「Localisation」を設定 (SSID / パスワード / 国 = JP / タイムゾーン = Asia/Tokyo)。
  8. 「Remote access」で SSH を有効化 (パスワード認証で可)。
  9. 各タブで設定が終わったら 保存。サマリー画面 (Write image) が表示されます。
  10. サマリーで「Customisations to apply」に Hostname / User / Wi-Fi / SSH が並んでいることを確認し、右下の赤い WRITE ボタンを押す。これで初めて SD に書き込まれます (v2 では「保存」だけでは書かれません)。
  11. 書き込み + 検証で 5〜10 分。完了したら microSD を一旦取り出し、もう一度 PC に挿し直してください。bootfs という名前のパーティションが見えるはずです。中に user-data というファイルがあれば成功。
  1. Open Pi Imager. Under Choose device, pick your Pi model.
  2. Under Choose OS → Raspberry Pi OS (other) → Raspberry Pi OS (64-bit) (full version recommended; Raspberry Pi OS Lite (64-bit) also works for a lighter footprint).
  3. Under Choose storage, pick the microSD card.
  4. Click Next. On the OS-customisation prompt, choose Edit Settings.
  5. Set Hostname:
    • Hostname = device name: s01, s02, ... (devices at SFC) or a01, a02, ... (devices in the API building). Must be unique per device. Enter in lowercase (DNS rules); first boot uppercases it into the device_id (S01 / A01).
  6. Set User:
    • Username and password: e.g. sodat. You'll re-enter this same username on the next step, so remember it.
  7. Set Wi-Fi and Localisation (SSID / password / country = JP / timezone = Asia/Tokyo).
  8. Under Remote access: enable SSH (password auth is fine).
  9. Click Save after each tab. Pi Imager shows a summary (Write image) screen.
  10. On the summary, confirm "Customisations to apply" lists Hostname / User / Wi-Fi / SSH, then click the red WRITE button. This is what actually writes the card — in v2 the Save button alone does not write anything.
  11. Write + verify takes 5–10 minutes. When it finishes, eject and re-insert the microSD on your PC. You should see a partition called bootfs containing a file called user-data.

ファイルとしてダウンロード Download as file

📂 ボタンは SD カードの bootfs フォルダをブラウザが直接読み書きします (user-data の末尾に追記)。ブラウザの権限ダイアログで「許可」を選んでください (このタブを閉じれば失効)。 📂 lets the browser read and append to the SD card's bootfs/user-data directly. Grant permission when the browser asks (revoked when this tab closes).

⚠ お使いのブラウザは 📂 直接書き込みに対応していません (Chrome / Edge / Safari 17.4 以降で対応)。📋 クリップボードにコピーを使い、bootfs/user-data の末尾に手動で貼り付けてください。 ⚠ Your browser doesn't support 📂 direct writing (supported in Chrome / Edge / Safari 17.4+). Use 📋 Copy to clipboard instead, and paste manually at the end of bootfs/user-data.

方法 A (📂 推奨, 1 クリック)

  1. microSD を PC に挿す (Mac なら自動で bootfs ボリュームがマウント)
  2. 上の 📂 ボタンをクリック → フォルダ選択ダイアログで bootfs を選ぶ
  3. ブラウザの「許可」を押す
  4. 「✓ SD に追記完了」を確認したら Step 4 へ

方法 B (📋 手動ペースト)

  1. microSD を PC に挿し、bootfs パーティションを開く
  2. user-data をテキストエディタで開く (Win: メモ帳、Mac: テキストエディット の標準テキストモード)
  3. ファイルの末尾に、📋 Copy で取得した snippet をそのまま貼り付ける (cloud-init は YAML なので、新しい write_files: と runcmd: ブロックを最後に並べる)
  4. 保存 (改行コードは LF のまま、CRLF に変えない) → Step 4 へ

Method A (📂 recommended, one-click)

  1. Insert the microSD into your PC (on macOS the bootfs volume mounts automatically).
  2. Click the 📂 button above → in the folder picker, select bootfs.
  3. Click Allow on the browser permission prompt.
  4. When you see "✓ Appended to user-data", continue to Step 4.

Method B (📋 manual paste)

  1. Insert the microSD; open the bootfs partition.
  2. Open user-data in a text editor (Notepad on Windows, TextEdit in plain-text mode on macOS).
  3. At the end of the file, paste the snippet you grabbed with 📋 Copy to clipboard. It contains two top-level YAML blocks (write_files: and runcmd:) that simply tack onto the existing document.
  4. Save the file (keep LF line endings, not CRLF), then continue to Step 4.
取り扱い注意: snippet (ダウンロードまたはコピーしたテキスト) にはプロジェクトの service_account.json が base64 で含まれます。チャットや公開リポジトリに貼らないでください。書き終わったら PC 上のダウンロードファイルも削除推奨。 Handle with care: the snippet embeds the project's service_account.json (base64). Don't paste it into chat or a public repo. Delete the downloaded copy from your PC after the SD card is written.

Pi に挿して電源 ONBoot the Pi

bootfs をアンマウント / 取り出して、microSD を Raspberry Pi に挿してください。電源を入れると初回起動が始まり、自動で: Unmount/eject bootfs, insert the microSD into the Raspberry Pi, and power on. The first boot will automatically:

  1. Wi-Fi に接続 (Pi Imager の設定)
  2. service_account.json を所定位置に展開
  3. GitHub からリポジトリを取得しインストール
  4. ホスト名から device_id を導出 (s05 → S05、a01 → A01)
  5. センサ収集サービスを起動
  1. Connect to Wi-Fi (from Pi Imager settings).
  2. Place service_account.json at the right location.
  3. Clone the repo from GitHub and run the installer.
  4. Derive device_id from the hostname (s05 → S05, a01 → A01).
  5. Start the sensor-collection service.

所要時間は 8〜15 分程度。インターネット回線次第です。 This takes ~8–15 minutes depending on your internet speed.

トラブルシュート Troubleshooting
  • 起動しても Google Drive にデータが来ない: /var/log/sodat-firstrun.log および cloud-init ログ /var/log/cloud-init-output.log を確認。journalctl -u sensor_sfc -n 200 も参照。
  • sNN.local / aNN.local で繋がらない: ルータの管理画面で IP を確認、または Pi にディスプレイを繋いで ip a で確認。
  • もう一度やり直したい: SD を PC で再フォーマットして Step 2 から。
  • Service running but no data in Drive: check /var/log/sodat-firstrun.log and the cloud-init log /var/log/cloud-init-output.log; also journalctl -u sensor_sfc -n 200.
  • Can't reach sNN.local / aNN.local: find the Pi's IP via your router, or connect a display and run ip a.
  • Want to start over: reformat the SD card on your PC and repeat from Step 2.
